## **Executive Summary of the Academic Consensus**

The trajectory of artificial intelligence and machine learning is rapidly approaching the physical and theoretical limits of the von Neumann architecture and the backpropagation of errors. As machine learning models scale to encompass billions of parameters, they increasingly encounter severe bottlenecks in memory bandwidth, power consumption, and real-time adaptability. The academic consensus from recent literature (2022–2026) unequivocally supports a structural paradigm shift toward continuous, unsupervised Hebbian learning embedded within non-matrix, neuromorphic architectures.1 This consensus is driven by the realization that biological neural networks do not rely on global error gradients transmitted through symmetric feedback connections. Instead, biological systems depend on local, asynchronous synaptic updates modulated continuously by real-time sensory streams, avoiding the computational halting required by traditional gradient descent.1

For autonomous, safety-critical edge environments—such as the intelligent cooperating vehicles envisioned in the KARYON (Kernel-based ARchitecture for safetY-critical cONtrol) project—the ability to learn continuously from streaming data without catastrophic forgetting or high-latency offline training is not merely an optimization; it is a strict operational necessity.5 The literature demonstrates that spatial pooling algorithms, specifically derivatives of the Hierarchical Temporal Memory (HTM) framework, successfully convert dense, noisy data streams into sparse distributed representations (SDRs) utilizing competitive Hebbian learning rules.6 These algorithms achieve rapid adaptation to changing input statistics and high fault tolerance entirely without relying on dense matrix multiplication.6

Simultaneously, a profound structural critique of traditional deep learning has emerged regarding data and knowledge representation. The reliance on static weight matrices and Generalized Matrix-Matrix Multiplication (GeMM) imposes an $O(n^2)$ scaling penalty and enforces rigid network topologies that cannot easily accommodate dynamic, temporally evolving data.9 In response, researchers are pioneering dynamic graph topologies where nodes represent discrete states, entities, or concepts, and edges represent temporally learned sequences generated via synaptic plasticity.11 Because these non-Euclidean data structures are memory-bound rather than compute-bound, their hardware implementation requires entirely novel approaches to concurrency and input/output (I/O) management.

To resolve the extreme I/O constraints and memory wall issues inherent in dynamic graph traversal, the academic community has increasingly turned to decentralized, message-passing software paradigms. Localized state machines governed by the Actor model (analogous to the Erlang/OTP BEAM VM) combined with zero-latency, brokerless message-passing protocols (such as ZeroMQ) provide a highly scalable, lock-free framework for asynchronous synaptic updates.13 This synthesis of Hebbian learning, dynamic graph topologies, and Actor-model concurrency offers a robust blueprint for next-generation neuromorphic hardware, addressing both the physical limitations of silicon and the adaptive demands of autonomous machine intelligence.

## **Theoretical Viability of Spatial Pooling on Raw Streams**

The biological brain continuously processes vast, high-dimensional streams of sensory information—translating variations in light luminance, sound pressure, and spatial deformation into millions of asynchronous spike trains.6 Understanding how individual cortical neurons learn to respond to these specific input patterns, and how populations of neurons collectively represent features in a flexible, dynamic, yet robust way, is the central objective of computational neuroscience.2

The dominant paradigm in artificial neural networks (ANNs), backpropagation, is functionally incompatible with this biological reality and introduces severe learning constraints. Backpropagation requires a complete forward pass to compute and store the activations of every neuron, a global halting of the network to calculate the error, and a sequential, synchronous backward pass that utilizes symmetric feedback connectivity to distribute precisely calculated gradients.1 In real-time streaming environments, this "stop-and-update" mechanism destroys temporal continuity, prevents parallelization during learning, and incurs massive memory overheads, making it fundamentally unsuitable for edge-deployed autonomous systems.1

In contrast, Hebbian learning postulates that synaptic strength increases when pre-synaptic and post-synaptic neurons activate simultaneously—a strictly local, asynchronous process encapsulated by the axiom "neurons that fire together, wire together".17 Recent advancements have formalized computational Hebbian learning for continuous, unsupervised feature extraction directly from raw data streams without objective functions or backward error propagation.4 The most prominent and empirically validated mechanism for achieving this is Spatial Pooling (SP), a core algorithmic component of the Hierarchical Temporal Memory (HTM) framework.6

### **The Mechanics of the Spatial Pooler**

The HTM Spatial Pooler acts as an algorithmic interface between raw, noisy input streams and higher-order temporal sequence memory.20 Its primary computational directive is to maintain a fixed-sparsity output (Sparse Distributed Representations, or SDRs) while preserving the semantic similarity of the input space.6 The Spatial Pooler achieves this through a continuous, unsupervised multi-step process involving overlap computation, competitive inhibition, and Hebbian synaptic updates.7

When a binary input vector is presented to the system, the SP computes the overlap between the active input bits and the connected synapses of each mini-column (a localized grouping of computational neurons).6 Only the mini-columns with the highest overlap scores—the "winners"—are allowed to activate, enforcing a strict winner-take-all competitive inhibition.7 Following this competitive selection, a Hebbian learning rule is applied exclusively to the active mini-columns: the permanence values (synaptic weights) of synapses connected to active input bits are increased, while the permanence of synapses connected to inactive bits is decremented.20 Over continuous iterations, this algorithm organically extracts the underlying spatial correlations of the data stream without requiring labeled data or offline batch processing.22

A crucial second-order algorithmic insight derived from the literature is the role of homeostatic excitability control, commonly referred to as "boosting." If a mini-column remains inactive for prolonged periods, the network artificially boosts its overlap score, ensuring that the entire computational capacity of the network is utilized and preventing a small subset of highly active neurons from dominating the representation.6 This provides an intrinsic fault tolerance and noise robustness that dense, matrix-based ANNs lack; if a portion of the spatial pooler is destroyed or corrupted (e.g., cell death or hardware failure), the boosting mechanism naturally and rapidly reallocates representational duties to adjacent healthy columns.6

| :-------------------------- | :---------------------------------------------- | :----------------------------------------------- |
| **Learning Paradigm**       | Supervised, gradient descent, global error      | Unsupervised, local competitive Hebbian          |
| **Data Ingestion**          | Static offline batches, sequential epochs       | Continuous online streaming data                 |
| **Memory Overhead**         | High (must store all forward activations)       | Low (updates are immediate and localized)        |
| **Fault Tolerance**         | Low (susceptible to catastrophic forgetting)    | High (homeostatic boosting reallocates capacity) |
| **Biological Plausibility** | Low (requires weight symmetry/weight transport) | High (mimics neocortical mini-columns)           |

### **Advancing Beyond Simple Hebbian Rules: The Information Bottleneck**

While traditional Hebbian learning is highly localized and efficient, it historically struggles with runaway synaptic potentiation and the scaling of complex tasks without some form of global constraint or feedback mediation.22 To address this, recent studies have introduced sophisticated variants, such as the Information Bottleneck (IB) based Hebbian learning rule.1

The IB principle balances two competing objectives for each layer's hidden representation: compressing the mutual information between the input and the representation, while maintaining enough information to preserve the relationship between the representation and the target output.1 By utilizing the Hilbert-Schmidt Independence Criterion (HSIC) as a proxy for mutual information, researchers have derived a three-factor learning rule that operates independently at each layer.1

The three factors include the local pre-synaptic activity, the local post-synaptic activity, and a global modulating signal.1 Because computing the global signal requires temporal context across multiple samples, researchers utilize an auxiliary reservoir network that acts as a working memory.1 This auxiliary network captures temporal dependencies across streaming data and modulates the local Hebbian updates, creating a profound, biologically plausible link between working memory capacity and continuous synaptic plasticity.1 Empirical data from recent implementations of this rule demonstrate test accuracy on par with backpropagation baselines on standard image classification tasks (e.g., MNIST and CIFAR-10) while maintaining effective batch sizes as low as 4 to 10 and utilizing neuron time constants of 5 ms.1

This localized, memory-modulated approach explicitly solves the primary critique of backpropagation: it trains each layer independently and asynchronously, entirely circumventing the weight transport problem and the need for backward error propagation across layers.1

## **Replacing Static Matrices with Dynamic Graph Topology**

The second fundamental divergence between conventional deep learning and advanced neuromorphic systems lies in data representation and topology generation. Modern machine learning frameworks (e.g., PyTorch, TensorFlow) process data almost exclusively via dense matrices and tensors, executing operations through Generalized Matrix-Matrix Multiplication (GeMM).9 While this maps efficiently to the highly parallel arithmetic logic units (ALUs) of traditional Graphics Processing Units (GPUs), it is fundamentally antithetical to the sparse, continuous, and dynamic nature of biological learning and streaming graph data.10

### **The Bottleneck of Dense Matrix Multiplication**

In standard Graph Convolutional Networks (GCNs), learning requires multiplying a sparse adjacency matrix by a dense feature matrix (SpMM), followed by a dense-dense matrix multiplication (GeMM) for weight updates.9 Profiling of single GPU GCN training reveals that up to 94% of the runtime is spent exclusively during the execution of the forward and backward SpMM kernels.9 As network sizes reach the scale of millions of vertices and billions of edges, these matrix operations encounter severe scalability critiques.9

The primary vulnerability of matrix-based learning is massive redundant communication and memory overhead.10 In a distributed system, executing an SpMM requires coarse-grained collective communication, such as SUMMA-like algorithms, which broadcast all local embeddings from one node to all others at each stage, regardless of the actual sparsity of the connections.10 Furthermore, during the forward pass of backpropagation, the activations of every layer must be stored in high-bandwidth memory (HBM) to compute the backward pass gradients, resulting in an $O(bLD)$ space complexity that rapidly exhausts hardware limits.16

More critically, static matrices cannot seamlessly accommodate dynamic topology. Real-world continuous data streams—such as financial transactions, biological interactions, or the sensor arrays of an autonomous Karyon vehicle—are Continuous-Time Dynamic Graphs (C-TDG).26 New nodes (entities) and edges (relationships) appear and disappear asynchronously in real-time.11 Recomputing, re-partitioning, or padding dense matrices to account for online topological changes is computationally paralyzing and prevents true inductive learning.12

### **The Shift to Dynamic Graph Representations**

To overcome these scalability limits, cutting-edge architectures reframe neural networks not as a rigid sequence of matrix multiplications, but as a dynamic, multi-relational graph topology where edges explicitly represent learned temporal sequences and correlations.12 In this paradigm, events are treated as asynchronous point processes, mapped as individual nodes with spatial and temporal coordinates.28

A prime example of this evolution is ChronoSpike, an adaptive Spiking Graph Neural Network (SGNN) tailored for dynamic graphs.11 Rather than relying on dense recurrent hidden states—which scale quadratically and consume massive memory—ChronoSpike maintains linear time complexity.11 It utilizes adaptive Leaky Integrate-and-Fire (LIF) neurons to encode spatial neighborhoods into sparse binary spikes, while a temporal integration module captures the evolving dependencies across sequential graph snapshots.11 By communicating strictly via sparse spikes rather than dense continuous vectors, the architecture achieves a dramatic reduction in both energy and memory costs.11

| Architectural Metric    | Dense Matrix Multiplication (GeMM/SpMM)         | Dynamic Graph Topology (e.g., ChronoSpike)        |
| :---------------------- | :---------------------------------------------- | :------------------------------------------------ |
| **Data Structure**      | Static dense tensors; padded sparse matrices    | Asynchronous event clouds; dynamic edge linking   |
| **Space Complexity**    | High $O(N \\cdot d)$ active memory per layer    | Low $O(E)$ memory; topology scales with activity  |
| **Temporal Processing** | Batch-processed sequences; rigid time steps     | Event-driven; inherently temporal point processes |
| **Communication**       | Global broadcasting; massive redundant data     | Point-to-point message passing (Spike events)     |
| **Adaptability**        | Requires full model retraining for new entities | Inductive; continuous addition of nodes/edges     |

The transition to dynamic graphs also draws deep theoretical inspiration from low-density parity-check (LDPC) codes and factor graphs used in error correction.27 In LDPC decoding, bipartite factor graphs utilize iterative message passing between variable nodes and factor nodes to refine estimates asynchronously.31 Applied to autonomous edge networks (such as IoT or cooperating vehicles), a dynamic graph treats physical entities (e.g., devices, sensors) as variable nodes and systemic constraints (e.g., interference budgets, collision thresholds) as factor nodes.31

The topology self-organizes without prior knowledge. Frameworks utilizing Dynamic Graph Topology Generation (DGTG) integrate the determination of the graph's structure directly into the end-to-end learning process.33 Relationships between multimodal sensory inputs are dynamically encoded as edges, allowing the system to extract complex, non-linear correlations through spatial pooling without ever instantiating a dense adjacency matrix.33

## **Managing I/O and Concurrency in Localized State Machines**

The transition from matrix-bound computation to dynamic graph topology inherently shifts the system's primary computational bottleneck. The architecture ceases to be compute-bound (starved for ALU floating-point operations) and becomes overwhelmingly memory-bound and I/O-bound.34 Traversing highly recursive, pointer-heavy graph structures across millions of nodes results in irregular memory access patterns, devastating cache locality, and immense synchronization overhead if managed poorly.36

To facilitate the continuous, asynchronous updates dictated by Hebbian learning, the software layer must be entirely decoupled from the synchronous, locking threads typical of standard operating systems. The academic consensus points to a synthesis of the Actor model of computation and zero-latency message-passing protocols as the definitive solution for decentralized neural architectures.13

### **The Actor Model and the BEAM VM**

The Actor model, originally conceptualized by Carl Hewitt, treats "actors" as the universal, isolated primitives of concurrent computation.38 An actor is an isolated state machine that can only affect its own internal state; it interacts with the broader system exclusively by passing and receiving asynchronous messages.38 In the context of a neuromorphic architecture, an actor perfectly maps to the biological concept of a single neuron or a cortical mini-column undergoing synaptic plasticity.14

The BEAM Virtual Machine, the runtime engine for the Erlang and Elixir programming languages, is the industry standard for implementing the Actor model.13 The BEAM VM provides preemptive scheduling, lightweight isolated processes (actors), and location transparency.13 Location transparency is critical for scaling decentralized neural architectures; it ensures that a "neuron" actor can send a spike (message) to a connected "synapse" actor without needing to know whether that synapse resides on the same CPU core, a separate GPU, or entirely different hardware across a distributed network.13

In highly distributed Federated Learning (FL) implementations such as Fed-lang, the BEAM VM is utilized to orchestrate the asynchronous training of neural networks without locking shared memory.13 By eliminating shared state, the Actor model fundamentally eradicates race conditions and the need for mutex locks (mutual exclusion).40 Every actor features its own isolated memory heap and its own independent garbage collection cycle, preventing the "stop-the-world" pauses that plague monolithic memory architectures.42 If a localized area of the graph topology experiences a high degree of stimulus (an I/O spike), those specific actors independently process their mailboxes without halting the rest of the neural network. This allows ultra-fine-grained concurrent processing that directly mirrors the asynchronous firing of a biological brain.14

### **Zero-Latency Message Passing with ZeroMQ**

While the Actor model provides the logical framework for concurrency, the physical transport layer must minimize the latency of these millions of asynchronous synaptic updates. Conventional TCP/IP overhead and centralized message brokers often succumb to flow control issues or memory exhaustion under high-throughput neural workloads.44 To resolve this, researchers leverage ZeroMQ (ZMQ), a high-performance asynchronous messaging library that provides a Socket-style API but entirely bypasses standard broker-based middleware.45

ZeroMQ operates as a decentralized, peer-to-peer transport layer.46 It natively supports patterns essential to neural architectures, such as Publish-Subscribe (analogous to a neuron broadcasting an action potential to multiple distal dendrites) and Push-Pull pipelines (for distributing computational workloads).46 ZeroMQ guarantees in-order message delivery and manages its own lock-free background I/O threads, allowing the main application to remain unblocked.45

In empirical evaluations of real-time neural decoding platforms, such as the Backend for Realtime Asynchronous Neural Decoding (BRAND), architectures utilizing ZeroMQ and in-memory data structures achieved sub-millisecond latencies—specifically <600 microseconds when transmitting 1024 channels of 30 kHz neural data.15 Furthermore, implementations using heterogeneous reinforcement learning architectures (HeteroRL) rely on ZeroMQ to transmit inference trajectories from samplers to learners, tolerating temporal delays and unpredictable network latency via lock-free parameter updates.48

By combining the BEAM VM's state isolation with ZeroMQ's decentralized transport, a dynamic graph can undergo continuous Hebbian learning in a completely lock-free environment.15 Algorithms like Hogwild! and modern lock-free bounded buffer synchronization mechanisms (BBS) ensure that local gradient updates or structural plasticity changes occur immediately as data streams in, achieving sub-millisecond trajectory learning with minimal overhead.50

| Concurrency Model             | State Management                                     | I/O Blocking & Latency                | Suitability for Neuromorphic Graphs         |
| :---------------------------- | :--------------------------------------------------- | :------------------------------------ | :------------------------------------------ |
| **Shared Memory / Mutex**     | Global shared state; prone to race conditions        | High blocking; threads wait on locks  | Low; deadlocks scale with graph size        |
| **Centralized Broker (AMQP)** | Managed by central server/queue                      | Moderate; broker becomes a bottleneck | Low; inadequate for massive parallel spikes |
| **Actor Model (BEAM) + ZMQ**  | Isolated local state per actor; location transparent | Zero blocking; asynchronous mailboxes | High; mimics biological neural independence |

## **Hardware Constraints: Memory-Bound vs. Compute-Bound Learning**

The theoretical elegance of continuous Hebbian learning on dynamic graphs is ultimately constrained by the physical properties of silicon. Conventional computing hardware is optimized for the von Neumann architecture, where data is continuously shuffled between separate processing (CPU/GPU) and memory (DRAM) units over a narrow bus.

### **The Memory Wall and I/O Bottlenecks**

For dense matrix multiplication, operations are heavily compute-bound; once a block of the matrix is loaded into the processor's SRAM or L1 cache, the ALUs perform thousands of multiply-accumulate (MAC) operations on it before needing to fetch more data from main memory.53 However, dynamic graph traversal, spatial pooling, and asynchronous synaptic updates are notoriously memory-bound.35 Finding the neighbors of a node requires pointer chasing through scattered memory addresses, leading to continuous cache misses.36 Furthermore, the reduction operations necessary for integrating spike data (e.g., gather-reduce-scatter) rely almost entirely on memory bandwidth rather than floating-point logic.35

Modern hardware attempts to mitigate this through advanced packaging, such as High-Bandwidth Memory (HBM) utilizing 8-channel DDR4/DDR5 configurations or TSV (Through-Silicon Via) stacked DRAM.55 An 8-channel unified memory architecture provides dedicated memory bandwidth spanning hundreds of gigabytes per second, theoretically supporting the rapid read/write cycles required by Hebbian synaptic updates.55 However, even 8-channel ECC RAM caching limits are quickly exhausted by the vast size of large-scale dynamic graphs (often scaling to trillions of edges) if the data is not partitioned meticulously.59 Because ECC (Error-Correcting Code) RAM introduces slight latency overheads to verify data integrity, the constant random access patterns of graph traversal severely penalize overall throughput.

### **Neuromorphic Architecture and Lock-Free Optimizations**

To truly overcome the memory wall, the hardware must physically mimic the algorithmic topology. Next-generation neuromorphic architectures are abandoning standard CPUs and GPUs in favor of Processing-In-Memory (PIM) and Processing-Using-Memory (PUM) technologies.35 By embedding logical units directly inside the DDR DIMMs, or by utilizing non-volatile nanoscale electronic switching devices like Memristors (memory resistors) arranged in crossbar arrays, the architecture eliminates the von Neumann data transfer penalty entirely.35 Memristors intrinsically perform analog computation within the memory cell itself, offering an ideal, highly energy-efficient hardware substrate for storing continuously variable Hebbian synaptic weights.33

In the software simulation of these future hardware states, extreme optimizations are required to prevent I/O starvation. Hybrid CPU-GPU simulators, such as Simeuro, demonstrate that simulating multi-core neuromorphic chips at scale requires highly orchestrated, asynchronous, stall-free pipelines.62 Simeuro implements lock-free parallelization algorithms for the routing component on the CPU to handle the fine-grained details of Network-on-Chip (NoC) spike communications without triggering race conditions.62

By completely removing mutex locks and relying on atomic operations (such as compare-and-swap) or localized state queues (like the Actor model mailboxes), the hardware can fully saturate its memory bandwidth with graph updates rather than idling in blocked thread queues.52 This holistic alignment—where the Hebbian learning algorithm, the dynamic graph data structure, the Actor-model concurrency, and the memory-centric hardware all operate without global synchronization—forms the only viable path forward for hyper-scaled, autonomous AI systems.
